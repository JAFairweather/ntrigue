## 1. What Ntrigue is

A party game of **secrets, prisoner's-dilemma trades, and blackmail** for 3–6 friends around a dinner table, played entirely on phones (with an optional TV "stage" screen). Built from a spec (`ntrigue-spec.md`, originally on Google Drive) on top of James's own **NIP-DA Scoped Data Grants** nostr protocol (`github.com/JAFairweather/nostr-scoped-data-grants`).

**Core loop:** Four rounds, then a finale. Each round: everyone answers a prompt with a true secret (encrypted, held only by them) → each player is matched with one other player → both secretly pick SHARE or HOLD (commit–reveal). Payoffs: both share = 3 points each and you read each other's secrets privately; one shares/one holds = holder gets 5 + the sharer's secret + a public 🗡 marker, sharer gets 1; both hold = 1 each. **Finale** (one move each, lowest score first): pick a collected secret and either EXTORT its owner ("pay 3 points or it goes public" — pay transfers 3; refuse lets the extorter reveal for +2 or fold), read it out to the room (+1), or keep quiet (+2; auto-applied if you collected nothing). Most points wins; side awards: Villain of the night (most 🗡) and Biggest sucker (most betrayed). Design law: *you can revoke a secret, but you can't un-tell it.* Key strategic fact confirmed in discussion: each specific secret has exactly one collector (only your round-match can obtain it), so a secret exposed publicly can never be replayed — the move that makes it public also spends it. Private = valuable, public = worthless.

## 3. Architecture (all vanilla ESM, no build step, no server)

- **Transport:** public nostr relays are the game room (`relay.damus.io`, `nos.lol`, `relay.primal.net`; override with `#r=<url,url>` in the address). `net.mjs`: SimplePool wrapper (publish with 8s ack race, query with 4s maxWait, subscriptions), `sendAction` (kind 30078 / NIP-78 app-data events, d-tags `ntg:<gid>:<suffix>`, t-tag = gid), room-code lookup via `c` tag `ntg-<CODE>`.
- **Protocol:** `nipxx.mjs` vendored unchanged from the NIP-DA repo. Each answer is a kind-30440 encrypted scope under its own key; a trade delivers a kind-440 gift-wrapped (NIP-59) grant of that key to the counterpart. Relays only ever see ciphertext, wraps, and public state (proven by adversarial tests).
- **Host-authoritative pure reducer:** the creator's phone folds player action events through `reduce(state, act, content)` in `state.mjs` (deterministic, seeded RNG via sha256, no Date/Math.random in reducer) and publishes public state (d = `ntg:<gid>:state`). Players just render the latest state. `state.promptText` carries drawn prompt text so clients never need the full deck (enables host-local AI decks).
- **Commit–reveal** for SHARE/HOLD: `sha256(choice+':'+nonce)`.
- **Pairings** (`pairingsFor`): 4 players → R1 cross (1,3)(2,4), R2 partners (1,2)(3,4), R3 other cross, R4 = R2 (couples sit 1+2, 3+4); other counts use circle method; warm-up round uses R1's pattern.
- **Files:** `state.mjs` (engine), `net.mjs`, `app.mjs` (~1000 lines: phone client + host driver + bots), `tv/tv.mjs` + `tv/index.html` + `tv/tv.css` (stage), `copy.mjs` (ALL player-facing strings + `MC_UI` (exempt) + `BOT`), `deck.json`, `quips.json`, `mc.mjs` + `mc-policy.md` + `proxy/worker.js` + `mc.json.example` (AI MC), `style.css`, `index.html`, `about.html` (protocol credit, scan-exempt), `vendor/nostr-tools.js` (esbuild bundle: nostr-tools 2.x + noble sha2 + qrcode-generator), README.md, tests under `test/`.
- **Language rule (hard):** zero protocol vocabulary anywhere a player can see — banned words include nostr/relay/scope/grant/key/event/publish/sign/encrypt/npub etc. — enforced by `test/banned-words.mjs` which scans the `UI` export, `BOT` export, all deck prompts (all flavors + practice), all quips, and index.html visible text. `MC_UI` and about.html are deliberately exempt.

## 4. Major features, in the order built

1. **v0 game** (PR #1 era): full 4-round + finale game, 40-prompt deck, quips, refresh-rejoin via localStorage + latest state, drama beats, host bar controls (Everyone's in / Redraw / Resolve-stragglers-hold / Next).
2. **TV Stage (M-TV):** `/tv` read-only display client with throwaway identity; 4-letter room codes (derived from gid, ambiguity-free alphabet); stage_join/stage_ping(20s)/stage_gone(65s stale watchdog); 8 rule-based style archetypes (Open Book, Vault, Shark, Mark, Diplomat, Anarchist, Enforcer, Wildcard — newest-earned tie-break) shown on stage scoreboards; WebAudio-synthesized stingers (host-toggled, off by default); phones become controllers with a 📺 chip when stage present; privacy by construction (stage key receives zero grants — tested).
3. **AI Master of Ceremonies (M-MC):** `mc.mjs` single interface (generateDeck / liveQuip 2.5s budget / closingRoast); three keyless-first backends: Instant (free `text.pollinations.ai`), site proxy (Cloudflare Worker `proxy/worker.js` holding ONE `claude-haiku-4-5` key server-side, pinned policy/schemas — NOT yet deployed, needs James's Cloudflare account + key, instructions in README), and advanced BYOK. Static deck/quips are the permanent fallback; the game never stalls on AI. MC inputs are public-state-only (privacy proven in tests). James's explicit direction: **no host-typed API keys, very low cost model**.
4. **How-to-play instructions** (PRs #6, #8, #10, #11, #15, #16, #18 — heavily iterated on James's feedback, see §5).
5. **TV reconnect** (PR #7): `/tv/#CODE` in the address always beats the remembered table (host phone shows this exact link); persistent "↩ New table" corner button on every stage screen; finished/missing/12h-cold remembered nights auto-forgotten on revisit.
6. **Lock-in resilience** (PR #12) after the first real game failed (only the host could lock an answer): taps never wait on the network (store-first, instant "Locked" UI + "Reaching the table…" until confirmed), delivery retried every 4s/state-change/foreground, sealed-copy and answered-flag are independent debts, poll fallback every 8s.
7. **Warm-up round + self-healing + clarity** (PR #13) after game night #1 feedback: **round 0 practice** (host toggle, on by default; mild prompt pool; full loop with gold coaching lines; debrief card; wipes scores/daggers/counters/collected before round 1); **self-healing transport** (the always-present state event is a liveness probe — 3 consecutive misses = zombie sockets → tear down and rebuild the whole pool + resubscribe, on phones, host, and TV; visibilitychange flush; goal: zero manual reloads ever); clarity fixes ("True secrets only", payoff numbers on the choice screen, "Your match this round: {name}", no more "partner" ambiguity).
8. **Deck flavors** (PR #14): lobby "Tonight's menu" — **Innocent** (default), **Spicy** (original deck), **Scorching**; `deck.json` v2 with `flavors` array; `state.flavor` set by start action (bad values → mild); both clients resolve names/prompts through `flavorRounds()`; AI decks bypass flavors.
9. **Robot guests** (PR #17): host lobby "+ Add a robot guest" (up to 5; names like Pixel 🤖, Nova 🤖). Bots are real players to the engine — own identities, actions through the same reducer via hostApply, secrets via real publishScope/grants — so solo play exercises the whole pipeline including the human privately reading a bot's secret and being extorted by one. Behavior: canned funny one-liners (in `copy.mjs BOT`, scanned), SHARE 70%, hand over sealed copies after resolution, fetch their own stash (needed to burn/reveal with real words), finale: 45% extort / 25% burn / rest vault; pay-refuse 50/50; reveal 60%. Delays 0.8–2.6s. 1 human + 2 bots = playable night.
